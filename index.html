TDDはテストではない

多分今日既に誰かがいったはず

TDDでコーディングしてる時たしかにテストしている感覚はない

チームで考えた時はどうだろうか

「そこはテスト書いてあるので大丈夫です」

ある程度TDDに理解ある人でも実際にコード書かない人(マネージャとか)はClean Code That Worksよりテストで動作が保証されていることを期待してる感じがする

そもそもテスターがいないチームではTDDで書いたユニットテストをテスト資産としないと手がまわらない

動詞のテスト
名詞のテスト

動詞のテスト=テストをする=問題を探す

名詞のテスト=テストコード(それだけじゃないけど)

TDDは動詞のテストではないが、名詞のテストを生み出している

テストではないと言い切ったときのもやもや感

重要なのは何であるかではなく、何をするか

振舞い!

結局はプログラミングなんですから必要だと思うことをやればいいと思いますよ！
